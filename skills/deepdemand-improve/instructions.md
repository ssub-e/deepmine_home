# Skill: deepdemand-improve (자동 품질 검증 및 하드닝 워크플로우)

본 스킬은 변환 단계([Transform Phase](../deepdemand-transform/instructions.md))를 거쳐 1차 구현이 완료된 소스코드를 정밀하게 분석하고, 규격 준수 여부를 채점하여 **기준 점수(95점 이상)를 통과할 때까지 소스코드를 스스로 교정(Hardening)하는 자동화 루프 스킬**입니다.

---

## 🔄 1. "Desloppify" 품질 검증 루프 (Desloppify Loop)

```
[1. SCAN] ──➔ [2. SCORE] ──➔ [3. PRIORITIZE] ──➔ [4. FIX] ──➔ [5. VERIFY]
   스캔            채점           우선순위 설정         수정          재검증
```

1. **스캔 (Scan)**: 대상 소스코드 파일 및 빌드 산출물을 읽고 미흡한 마크업 및 디자인 규격 위반 사항을 검출합니다.
2. **채점 (Score)**: 표준 채점표(Score Card)에 따라 항목별 감점을 적용하여 최종 점수를 산출합니다.
3. **우선순위 설정 (Prioritize)**: 감점 요인 중 기능 장애 및 접근성 위반 사항을 최우선 수정 대상으로 지정합니다.
4. **수정 (Fix)**: 감점된 항목을 해결하기 위해 정밀한 코드 교체(Replace)를 적용합니다.
5. **재검증 (Verify)**: 수정된 코드를 빌드(`npm run build` 등)하여 에러가 없는지 확인하고 스캔을 다시 돌려 최종 점수를 갱신합니다. 점수가 95점 이상이 될 때까지 이 과정을 반복합니다.

---

## 📊 2. 디자인 시스템 표준 채점표 (Score Card)

최종 점수는 **100점 만점**에서 시작하여 각 위반 사항별 감점(Deduct) 방식으로 계산합니다. 합격 기준은 **95점 이상**입니다.

### 2.1. 색상 체계 및 테마 (최대 20점)
*   **하드코딩 컬러 사용 (감점 -5점 / 건당)**: CSS 변수를 거치지 않고 인라인 헥스코드나 테일윈드 빌트인 컬러(예: `text-slate-400`, `bg-[#0B1120]`)를 하드코딩한 경우.
*   **다크모드 선택자 누락 (감점 -5점)**: 라이트/다크 모드 변환 시 클래스 스위칭 구조(`.dark`)가 반영되지 않았거나 HSL 대응 변수가 지정되지 않은 경우.

### 2.2. 타이포그래피 규칙 (최대 20점)
*   **표제어 폰트 규칙 위반 (감점 -5점 / 건당)**: 메인 헤드라인에 Cormorant Garamond / Tiempos Headline 계열의 세리프(`font-serif`)가 적용되지 않았거나, 자간 조정 클래스(`tracking-[-0.02em]` 등)가 없는 경우.
*   **세리프 폰트 볼드화 (감점 -3점 / 건당)**: 디스플레이 세리프 폰트에 볼드 가중치(`font-bold`, `font-extrabold`)를 주어 두껍게 묘사한 경우 (DDDS/Claude는 regular/medium 400~500 고수).
*   **데이터 폰트 오류 (감점 -3점)**: 과거 이력 연도, 수치, 차트 데이터에 고정폭 폰트인 JetBrains Mono(`font-mono`)가 빠진 경우.

### 2.3. 스페이싱 및 레이아웃 (최대 15점)
*   **모서리 곡률 규격 미준수 (감점 -3점 / 건당)**: 버튼(`rounded-sm`), 위젯 카드 및 아코디언(`rounded-lg`), 모달 및 일러스트(`rounded-xl`) 등의 표준 스케일을 벗어난 인라인 곡률(예: `rounded-3xl`)을 선언한 경우.
*   **섹션 간격 불일치 (감점 -5점)**: 메인 페이지의 상하 섹션 분할 여백이 `py-24` (96px) 혹은 `spacing.section` 규격을 지키지 않고 촘촘하거나 과하게 넓은 경우.

### 2.4. 웹 접근성 및 시맨틱 HTML (최대 25점)
*   **명도 대비 4.5:1 미달 (감점 -10점 / 건당)**: 옅은 회색 글씨 등 라이트 모드나 다크 모드 환경에서 텍스트와 배경의 명도차가 4.5배 미만이 되는 조합을 사용한 경우. (특히 `#94A3B8`을 라이트모드 흰색 배경 위에 쓴 경우 심각한 위반).
*   **레이블-인풋 바인딩 누락 (감점 -5점 / 건당)**: `<label htmlFor="...">`와 `<input id="...">`가 유기적으로 매핑되지 않은 경우.
*   **대체 텍스트 누락 (감점 -5점 / 건당)**: 딥마인 로고 이미지나 상태 표시용 스피너/그래픽 요소에 알맞은 `alt` 속성 혹은 `aria-hidden="true"` 처리가 누락된 경우.
*   **포커스 링 누락 (감점 -5점)**: 대화형 요소(인풋, 버튼)에 포커스가 갔을 때 아웃라인 링(`focus:ring-2 focus:ring-blue-500`) 표시가 해제되어 보이지 않는 경우.

### 2.5. 상태 처리 및 스크립트 결함 (최대 20점)
*   **서버 연동 폼 상태 제어 미흡 (감점 -5점)**: 도입 문의 폼 제출 중(`loading`)일 때 제출 버튼이 비활성화(`disabled`)되지 않거나 중복 제출 방지 처리가 없는 경우.
*   **모달 포커스 트랩(Focus Trap) 부재 (감점 -5점)**: 모달 창이 열렸을 때 Tab 키 포커스가 닫힌 상태인 뒷배경으로 새어 나가는 경우.

## 🛠️ 3. 자율 피드백 및 자가 교정 프로토콜 (Auto-Fix Protocol)

검증 스크립트 혹은 AI 에이전트는 감점 사항이 발견되면 아래 동작 매뉴얼에 따라 소스코드를 자동으로 교정합니다.

1. **로그 생성**: 검출된 위반 코드가 몇 번째 줄(Line)에 어떤 요소를 포함하고 있는지 기록합니다.
   * *예*: `[DEDUCT -5] App.jsx:L107: bg-[#0B1120] 사용 - 하드코딩 컬러 감점`
2. **패치 코드 생성**: 디자인 명세서([DESIGN.md](../../DESIGN.md))의 토큰에 맞춘 코드 교체 척도를 설계합니다.
   * *예*: `bg-[#0B1120]` ➔ `bg-background`
3. **정밀 수정 도구 사용**: `replace_file_content` 또는 `multi_replace_file_content` 도구를 사용해 해당 소스코드를 정확히 수정합니다.
4. **리빌드 및 테스트**: 수정이 완료되면 `npm run build` 명령을 실행해 정적 타입 빌드 및 포스트 프로세서(PostCSS) 컴파일 과정에서 오류가 발생하지 않는지 체크합니다.

## 📝 4. 보고서 템플릿 규격 (Improver Report Spec)

교정을 마친 후 최종 검증 단계에서 AI 에이전트는 아래 보고서 서식을 채워 터미널이나 에디터에 저장해야 합니다.

```markdown
# [Improve Report] DeepDemand Design Hardening

- **대상 파일**: `src/App.jsx`, `src/index.css`
- **검증 일시**: 2026-06-08 (최종 통과 시각)
- **최종 점수**: 98 / 100 점 (합격)

## ❌ 교정 전 감점 사항
1. [L107] `bg-[#0B1120]` 하드코딩 컬러 검출 (감점 -5)
2. [L256] 헤드라인 `font-extrabold` 볼드체 사용 (감점 -3)

## ✅ 교정 후 조치 내역
1. [L107] `bg-background`로 CSS HSL 변수 연동 완료.
2. [L256] `font-serif font-normal tracking-tight`로 에디토리얼 타이포그래피 준수 완료.
3. 빌드 테스트 결과: 성공 (`npm run build` 정상 통과)
```
