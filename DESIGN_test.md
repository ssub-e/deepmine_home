# DeepDemand 디자인 시스템 명세서 (DESIGN_test.md)
*이 문서는 DeepDemand B2B SaaS 대시보드와 향후 회사 홈페이지의 디자인 시스템을 통합하기 위한 설계 초안입니다.*

---

## 🏗️ 1. 디자인 철학 및 원칙 (Design Principles)

1. **Zero-Friction (초저마찰 사용자 경험)**
   * 복잡한 차트나 무거운 데이터 테이블에서도 사용자가 헤매지 않고, 가장 중요한 **'내일 추천 발주량'**과 **'위험 신호(품절/악성재고)'**를 3초 이내에 시각적으로 파악할 수 있어야 합니다.
2. **Premium & Intelligent (신뢰감을 주는 프리미엄 디자인)**
   * 일반적인 원색의 블루/레드 대신, 세련된 HSL 기반의 테일러드 컬러 팔레트와 둥근 모서리(Border Radius), 부드러운 트랜지션 애니메이션을 사용해 프리미엄 B2B 서비스의 신뢰감을 전달합니다.
3. **Data-Driven Clarity (데이터 시각화의 명확성)**
   * 시계열 차트, 기여도 그래프(SHAP) 등 핵심 정보를 표시할 때, 차트 색상이 브랜드 색상과 섞이지 않도록 대비가 분명하고 정제된 데이터 전용 컬러 스펙트럼을 사용합니다.

## 🎨 2. 색상 시스템 (Color Palette)

본 프로젝트는 `globals.css`를 기반으로 한 CSS Variables(HSL)와 Tailwind CSS의 하이브리드 바인딩 모델을 준수합니다.

### 2.1. 기본 테마 컬러 (Core Palette)
| 구분 | Light Mode (값) | Dark Mode (값) | 대표 색상 코드 | 용도 및 시각적 의도 |
| :--- | :--- | :--- | :--- | :--- |
| **Primary** | `221 83% 53%` | `221 83% 53%` | `#1D4ED8` (Deep Blue) | 메인 브랜드 컬러. 인공지능과 신뢰감 상징 |
| **Accent** | `191 78% 64%` | `191 78% 64%` | `#5ACAE8` (Vibrant Light) | 차트 포인트, 중요 수치 강조, 활력 부여 |
| **Background** | `210 20% 98%` | `210 10% 11%` | `#F8FAFC` / `#1A1D20` | 대시보드 캔버스 배경 (다크모드는 저자극 다크그레이) |
| **Card** | `0 0% 100%` | `210 10% 14%` | `#FFFFFF` / `#202428` | 데이터 시각화 위젯 및 차트 영역 카드 배경 |
| **Border** | `214.3 31.8% 91.4%` | `217 19% 27%` | Slate 계열 연한 경계선 | 카드 테두리, 구분선용 저자극 보더 |

### 2.2. 시맨틱 경고 신호등 컬러 (Status & Alerts)
B2B 재고 분석 화면에서 직관적인 인지를 돕는 3색 경고 시스템입니다.
*   🔴 **품절 임박 및 품절 리스크 (Alert/Danger):** `HSL 0 84% 60%` (Vibrant Red)
    *   *적용 예시:* 리드타임 내 재고 소진 가능성이 높은 품목의 발주 유도 경고등
*   🟡 **악성 재고 리스크 (Warning):** `HSL 35 92% 50%` (Vibrant Orange/Amber)
    *   *적용 예시:* 판매 속도 대비 보관 일수가 너무 긴 과적재 재고 경고등
*   🟢 **정상 및 적정 재고 (Safe/Normal):** `HSL 142 76% 36%` (Vibrant Green)
    *   *적용 예시:* 적정 안전 재고를 유지 중인 정상 상태 항목

##  font-family 3. 타이포그래피 (Typography)

*   **서체 조합:** 
    *   영어/숫자 가독성을 위해 **Inter**를 우선 적용하고, 한글 가독성 최적화를 위해 **Pretendard**를 결합하여 사용합니다.
*   **글꼴 크기 스케일 (Base: 16px = 1rem):**
    *   `text-xs` (12px / 0.75rem) - 차트 범례, 데이터 테이블 보조 지표
    *   `text-sm` (14px / 0.875rem) - 기본 본문 텍스트, 테이블 데이터, 버튼 라벨
    *   `text-base` (16px / 1rem) - 본문 강조, 인풋 레이블
    *   `text-lg` (18px / 1.125rem) - 위젯/카드 서브 타이틀
    *   `text-xl` (20px / 1.25rem) - 대시보드 위젯 메인 요약 수치
    *   `text-2xl`~`text-3xl` (24px~30px) - 온보딩/리포트 결과 메인 헤드라인

## 📐 4. 레이아웃 & 스페이싱 가이드 (Layout & Border Radius)

*   **Border Radius (모서리 곡률):**
    *   글로벌 공통 테마 값: `var(--radius) = 0.6rem` (9.6px) 적용.
    *   대시보드의 카드는 부드럽고 친근한 인상을 주는 동시에, 너무 유치하지 않도록 `0.6rem` 값을 카드와 모달(Dialog) 창의 표준으로 지정합니다.
*   **스페이싱 스케일 (Grid & Padding):**
    *   컴포넌트 내 여백: `p-4` (16px) 또는 `p-6` (24px) 단계를 유지하여 충분한 여백(Whitespace)을 확보합니다.
    *   페이지 그리드: `gap-6` (24px)를 기본 대시보드 그리드 간격으로 설정하여 위젯 간 구분을 명확히 합니다.

## 📊 5. 차트 및 시각화 디자인 규칙 (Chart Design)

Recharts를 활용한 예측 데이터 시각화 시, 테마에 구애받지 않고 일관된 가독성을 보장하기 위해 아래 차트 색상 스케일을 고수합니다.

1.  **과거 판매 데이터 (Historical Sales):** 톤다운된 저채도의 그레이-블루 계열 (`#94A3B8`)
2.  **AI 예측 수요선 (AI Forecast Demand):** 브랜드 메인 컬러인 블루 계열 (`#1D4ED8` 또는 `#3B82F6`)
3.  **안전 재고 기준선 (Safety Stock Level):** 명확한 기준선을 나타내기 위한 대조적인 점선 스타일
4.  **SHAP 변수 기여도 (XAI Contributions):**
    *   수요 상승 요인: 연한 파스텔 블루 (`#60A5FA`)
    *   수요 하락 요인: 연한 파스텔 로즈/그레이 (`#F87171` 또는 `#CBD5E1`)

## 🔗 6. 회사 홈페이지 통합 시 디자인 조율 로드맵

향후 회사 홈페이지와 본 솔루션을 병합하거나 서브도메인으로 연결할 때, 아래의 순서에 따라 스타일 정의를 동기화합니다.

1.  **CSS Variable 이름 및 HSL 값 매핑:**
    *   회사 홈페이지에 적용될 고유 브랜딩 컬러를 획득한 후, `globals.css`의 `:root` 내 `--primary` 및 `--accent` 색상 값을 홈페이지 가이드라인 수치와 맞춤으로써 브랜딩 통합을 완료합니다.
2.  **Tailwind 설정(tailwind.config.ts) 공유:**
    *   모노레포로 병합하거나 두 프로젝트를 단일 레포지토리로 통합할 경우, 하나의 `tailwind.config.ts` 파일에서 공통 테마(색상명, 폰트, 둥글기 등)를 통합 선언하여 양쪽의 컴포넌트가 같은 테마를 상속받게 만듭니다.
3.  **다크/라이트 모드 지원 일치:**
    *   홈페이지와 솔루션 대시보드가 모두 `next-themes` 패키지를 이용한 클래스 기반 다크모드(`.dark`)를 사용하도록 규격화하여, 페이지 전환 시 사용자가 느끼는 시각적 피로도를 최소화합니다.
